** mount device

The Handshake (The "Trust" Dance)
Plug the iPad into your PC.

Unlock the iPad screen.

On your Linux terminal, run:

#+begin_src sh
idevicepair pair
#+end_src

Look at the iPad: A "Trust This Computer?" prompt will appear. Tap Trust and enter your iPad passcode.

Run idevicepair pair again to confirm. It should say SUCCESS.

Mount the Filesystem
You need to create a mount point and then use ifuse to map the iPad's media partition to it.

Create a mount point in your home dir

#+begin_src sh
mkdir -p ~/mnt
#+end_src

Mount the media folder (Photos, Books, etc.)

#+begin_src sh
ifuse ~/mnt
#+end_src

You can now cd ~/mnt or open your file manager (Nautilus, Dolphin, Thunar).

Photos/Videos: Look in DCIM/100APPLE.

App Files: If you want to access files inside a specific app (like VLC or a PDF reader), use:

#+begin_src sh
ifuse --documents <appid> ~/mnt
#+end_src

(You can find the <appid> by running ifuse --list-apps).

Permissions: If you get a "Permission Denied" error, ensure your user is in the plugdev or adbusers group, or just run the mount command with sudo (though ifuse usually prefers being run as a user).

The "Dirty" Mount: If the iPad disconnects, the mount point will "ghost." Unmount it forcefully with:

#+begin_src sh
fusermount -u ~/mnt
#+end_src
